# parseallnums

Учебный проект для исчерпывающего решения вопроса разбора
введённых пользователем чисел. А также пример монадического
комбинаторного парсера на Java, написанного полностью в
функциональном стиле.

## Особенности
- **Функциональный стиль** – без циклов и переменных, только чистые функции, монады и константы
- **Комбинаторный парсер** – грамматика через композицию функций, легко расширять
- **Абсолютная точность** – результат выдаётся как `BigFraction` (рациональная дробь) без потерь и округлений

## Возможности
| **Конструкция** | **Примеры** |
|----------------|-------------|
| **Целые числа** (основания 2, 8, 10, 16) | `123`, `-456`, `0b1010`, `0o755`, `0xFF`, `0x1A3` |
| **Дробные числа** через точку | `1.5`, `.25`, `10.`, `0b1.01`, `0o3.4`, `0x1.F`, `0x.A` |
| **Экспонента `e`/`p`** – десятичная и двоичная | `1e10`, `1e-10`, `0b1p3`, `0x1.8p+4`, `0o7p-2`, `.5e4` |
| **Разделители разрядов `_` и `'`** | `1_000`, `0xCAFE_BABE`, `0o7'5'5`, `1.5_0`, `0b1010_1010` |
| **Очень большие числа** | `100000000000000000000000000000000000000000000`|

## Цели и ограничения
Проект не ставит целью написание _production ready_ 
парсера. Вопросы производительности и _stack overflow_
из-за рекурсии оставлены в пользу функционального подхода.
Проект не ставит целью построение парсера выражений с
операторами и функциями.

Цель — поддержка множества вариантов записи чисел без
потери точности и демонстрация функционального программирования
на Java.

Приветствуется добавление новой функциональности, при условиях:
 * Это имеет математический или практический смысл
 * Не нарушается обратная совместимость по возможностям
 * Это представимо в `BigFraction` без потери точности
 * Не нарушается функциональный стиль кода или
    архитектура парсера
